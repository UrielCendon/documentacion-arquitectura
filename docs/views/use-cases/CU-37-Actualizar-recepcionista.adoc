== CU-37 Actualizar recepcionista
[cols="25,~",options="header"]
|===
| ID | CU-37
| Nombre | Actualizar recepcionista
| Autor | Uriel Cendón Díaz
| Fecha de actualización | 31 de octubre de 2025
| Descripción | El gerente actualiza datos de un recepcionista asociado a uno de sus hoteles.
| Actor(es) | Gerente
| Disparador | El gerente selecciona actualizar un recepcionista.
| Precondiciones | -PRE-01: El gerente está autenticado en el sistema. +
-PRE-02: El gerente tiene al menos un hotel relacionado. +
-PRE-03: El recepcionista existe y pertenece a uno de los hoteles del gerente.
| Flujo normal |
1.- El sistema muestra los hoteles del gerente y solicita filtros (nombre, apellidos, email). +
2.- El gerente elige el hotel y los criterios de búsqueda para los filtros. +
3.- El sistema valida el hotel y muestra el listado de recepcionistas coincidentes. (Ver FA 3.1) +
4.- El gerente selecciona un recepcionista. +
5.- El sistema muestra los datos actuales y solicita los campos a actualizar (apellidos, email, nombre, telefono, contraseña). (Ver FA 5.1) (Ver FA 5.2) +
6.- El gerente captura los cambios. +
7.- El sistema valida los campos, actualiza la información y registra la operación, así como muestra un mensaje de confirmación en la operación. (Ver EX 1) (Ver EX 2) +
8.- Termina el caso de uso.
| Flujos alternos |
FA 3.1 Sin resultados. +
1.- El sistema informa que no hay coincidencias y sugiere ajustar filtros. +
2.- Termina el caso de uso. +
FA 5.1 El gerente canceló antes de actualizar. +
1.- El sistema cancela la operación a petición del gerente. +
2.- Termina el caso de uso. +
FA 5.2 - Los nuevos datos ya están en uso por otro recepcionista. +
1.- El sistema informa que los datos ingresados están registrados en otro recepcionista actualmente. +
2.- Regresa al paso 1 del flujo normal. +
| Excepciones |
EX 1 - No se pudo actualizar el recepcionista. +
1.- El sistema informa error al actualizar y registra el incidente. +
2.- Termina el caso de uso. +
EX 2 - El sistema no puede conectarse a la base de datos. +
1.- El sistema muestra un mensaje de error solicitando intentar más tarde y registra el suceso. +
2.- Termina el caso de uso.
| Postcondiciones | POST-01: Los datos del recepcionista quedan actualizados. +
POST-02: Se guarda el registro de los cambios.
|Reglas de negocio|
